# HUD Telemetry Style Guide

## Purpose
Keep AFL UI robotic, readable, and operational.

## Style rules
- dark steel/charcoal foundations
- cyan, cool white, tactical green as primary data accents
- amber for warning
- red for critical failure
- technical typography, not playful display fonts
- motion should imply scan, lock, calibration, alert, or confirmation

## Required panels
- robot vitals
- structure/stamina/energy
- objective state
- score and round timer
- mode readiness (tackle/weapon state)
- directional alerts

## Validation
- no panel obscures objective state
- critical health/energy remains readable during collision
- UI reads like telemetry, not fantasy HUD
